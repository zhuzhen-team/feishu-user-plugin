## Brainstorm 阶段已锁定的关键设计决策

**A.1 (WS 机器级)**:
- cursor 表示 = `{file, offset, version: 1}` (byte offset，不用 event_id)
- owner heartbeat = mtime utimes，15s 心跳 / 60s stale
- 非 owner = 30s lock poll 失效检测，事件读完全 lazy（不后台 stat events.jsonl）
- 7 项防御 R1-R7（保守 reset / defer-rotate + force-rotate / proper-lockfile / partial line repair / ENOENT 复位 / disk-full 降级到 v1.3.8 buffer）
- `manage_ws_status` 5 个 action: `info | reconnect | claim | rotate | reconfig`

**A.2 (active profile 跨进程)**:
- env `FEISHU_PLUGIN_PROFILE` 在 v1.3.9 失效（**纯 SSOT**），仅 bootstrap-only
- 全部 profile 关联状态都 invalidate（包括 resourceKey 缓存）
- per-event profile tag 写入 events.jsonl
- WS 切换走 γ heartbeat reactor（不用 force-claim 风暴）
- dispatcher stat 是 sync（10μs / call）

**A.3 (setup CLI)**:
- 4 行决策矩阵（fresh / auto-migrate / preserve / update）
- 已存在 default + --app-id 时**默认拒绝**，要 `--force` 或 `--profile <name>`
- `LARK_PROFILES_JSON` env 向后兼容并存（不 deprecate）

**A.4 (event types)**:
- credentials.json::profiles[*].events 字段（per-profile，可选）
- **不预先 hardcode** 事件类型清单（任意 string，SDK rejection → stderr warning + 跳过）
- 3 个修改路径：手动编辑 / `manage_ws_status(reconfig)` / `FEISHU_PLUGIN_EXTRA_EVENTS` env (仅 first-time bootstrap)

**B.1 (IMAGE)**:
- 元数据混合方案 D：`upload_image` 自动缓存 + `image_path` 自动 derive + 显式 width/height/mime + 缓存 miss 清晰报错
- `image-size@^2.0.0` 新 dep
- `~/.feishu-user-plugin/image-meta.json` 缓存 LRU 1000 entries

**B.4 (CARD)**:
- `via` 默认从 `"bot"` (v1.3.6) 翻成 `"user"` (v1.3.9)——这是 BREAKING-ish，CHANGELOG 必标
- 抓包 3 路径：i 转发 → ii admin page → iii 协议层注入

**B 系列共同**:
- 实施 = 反向工程，capture 是 plan 一部分，不预设 fields
- 每类 ≥3 张 capture 交叉验证
- B.1 / B.4 互相独立；任一失败单独 ship 另一个；失败的推 v1.3.10
- fixture 进 `tests/fixtures/captured/` 作 round-trip regression

**D**:
- `feishu-docx@^0.7.0` 主路径，coverage 不够时手写 post-processor
- embedded resources 用 `feishu://image_token/<token>` 占位 URL，**不自动下载**
- 工具签名最小（仅 `document_id`，不加格式选项）
